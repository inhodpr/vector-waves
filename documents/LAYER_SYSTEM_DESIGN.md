# Extended Layer System, Extractor & OSM Integrations Design
*Revision 1 - Addressing Arch/UX, Memory Management, and Method Details*

## 1. Architectural Context & UX Considerations

### 1.1 Rasterization Resolution & Zoom (Dynamic Re-Rasterization)
Since OSM vectors are being rasterized to offscreen canvases to avoid massive vector overhead in `Zustand`, relying on a single static raster will cause severe pixelation when the global canvas is zoomed heavily.
**Strategy:**
- The `ImageLayerEntity` tracks its `rasterizedZoomLevel` (e.g., `1.0`).
- The `CanvasEngine` watches the global scale (`appState.timelineZoomLevel` or `appState.canvasScale`). 
- **Zoom In (Up-scaling):** When the user stops zooming (debounced by 300ms) and `currentGlobalScale > rasterizedZoomLevel * 1.5`, a background worker or non-blocking async `OffscreenCanvas` process is triggered to `reRasterizeOSMLayer()` at the new native resolution limit. 
  - *Constraint 1 (Max Zoom):* A hard limit `MAX_RASTER_ZOOM_LEVEL` (e.g., 10x) is enforced to prevent browser crashes from excessively large canvas allocations.
  - *Constraint 2 (Viewport Culling):* To avoid rendering massive unseen areas at high zoom levels, the `reRasterizeOSMLayer` must only rasterize the portion of the vector map intersecting the current visible canvas bounding box (`panX`, `panY`, viewport `width`, `height`).
- **Zoom Out (Down-sampling):** If the user zooms out significantly (`currentGlobalScale < rasterizedZoomLevel * 0.5`), the system will trigger a down-sampled re-rasterization to free up the massive memory blob and replace it with a lighter-weight asset.
- **Fail-safe:** While waiting for the high-res buffer, the engine smoothly scales up the low-res asset using standard 2D context transforms.

### 1.2 Transform Hierarchy & Coordinate Mapping
For the Extract Tool to locate the correct pixel in an `ImageLayerEntity`'s asset buffer from a screen mouse click, it must project down through two matrices:
1. **Global Transform Matrix:** The `(panX, panY, scale)` applied to the canvas viewport.
2. **Local Transform Matrix:** The `(x, y, scale)` properties of the specific `ImageLayerEntity`.

**Hit Testing Workflow:**
- **Screen $\rightarrow$ Canvas:** Calculate `canvasPos = (screenPos - globalTranslation) / globalScale`.
- **Canvas $\rightarrow$ Layer:** Loop through `ImageEntities` from top `zIndex` to bottom.
- For each layer, calculate `layerPos = (canvasPos - layer.translation) / layer.scale`.
- Check if `layerPos` is within `[0, layer.width]` and `[0, layer.height]`.
- Instead of using a global backbuffer, request the specific pixel from the layer's local memory to check `alpha > 0`. The first layer satisfying this is the target.

### 1.3 Asset Memory Management & Disk Caching
Generating multiple project-sized canvas contexts and keeping the original raw OSM vectors in React state will crash large projects.
**Strategy:**
- **Disk-Backed Vector Cache:** Instead of storing massive GeoJSON blobs in memory for future re-rasterization, the `ImageLayerEntity` will simply store the `cache_key` (the MD5 hash) generated by `map_processor.py`. 
- **On-Demand Loading:** When `reRasterizeOSMLayer` is triggered by a zoom event, the frontend will request the vector data directly from the local disk cache (`.osm_cache/<cache_key>.json`) via IPC, drastically saving RAM.
- **Compression:** All generated `OffscreenCanvas` layers must be converted to memory blobs using `canvas.convertToBlob({type: 'image/webp', quality: 0.8})`. The `appState` only stores the lightweight string from `URL.createObjectURL(blob)`.
- **Garbage Collection:** A strict cleanup routine to invoke `URL.revokeObjectURL(oldAssetId)` must be used whenever a layer is deleted or re-rendered.

## 3. Explicit File & Function Modifications

### 3.1 State Management Updates
**File:** `app/src/renderer/src/store/useAppStore.ts` (and `types.ts`)

- **New Type:** Add the `ImageLayerEntity` interface. 
- **Active Layer Tracking:** `activeLayerId: string | null;`
- **Actions:**
  ```typescript
  // Updates local matrix/crop of the selected layer
  updateLayerTransform: (id: string, transform: { x?: number; y?: number; scale?: number; crop?: { x: number; y: number; w: number; h: number }}) => void;
  
  // Garbage collects and removes layer
  deleteLayer: (id: string) => void; // MUST CALL URL.revokeObjectURL(assetId)
  
  // Triggers background re-render of OSM vectors to a new WebP blob, constrained by viewport and zoom limits
  reRasterizeOSMLayer: (
    id: string, 
    newZoomScale: number, 
    viewportBounds: { panX: number; panY: number; width: number; height: number }
  ) => Promise<void>;
  ```

### 3.2 Frontend OSM Integration Logic
**File:** `app/src/renderer/src/utils/osmImporter.ts` (New file combining DB bridging and render conversion)

- **Function:** Handles the output from the python `map_processor.py`.
  ```typescript
  /**
   * Generates compressed WebP blobs from raw OSM vectors via OffscreenCanvas.
   * @param geoJSON - The grouped output straight from the python bridge.
   * @param baseWidth - Base canvas width (e.g., 1080)
   * @param baseHeight - Base canvas height (e.g., 1080)
   * @param targetZoom - Multiplier for high-DPI scaling, capped at MAX_RASTER_ZOOM_LEVEL.
   * @param viewportBounds - The visible screen area to constrain rasterization, preventing memory overflow.
   * @returns A map of layer category to ObjectURLs. E.g., { 'transport': 'blob:...', 'water': 'blob:...' }
   */
  export async function rasterizeOSMLayers(
    geoJSON: any, 
    baseWidth: number, 
    baseHeight: number, 
    targetZoom: number,
    viewportBounds: { panX: number; panY: number; width: number; height: number }
  ): Promise<Record<string, string>>;
  ```

### 3.3 Hit Testing & Extraction Tool
**File:** `app/src/renderer/src/tools/LineExtractor.ts`
The extractor must evolve from global processing to localized matrix projection.

- **Function:** Ridge Analysis against isolated buffer.
  ```typescript
  /**
   * Evaluates the trace path strictly against the targeted layer's coordinate space.
   * @param globalPoints - Points drafted by the user in global canvas space.
   * @param layerAssetId - The Blob URL of the detected top-most layer.
   * @param layerTransform - The matrix properties to inverse-project globalPoints down.
   */
  public static async analyzeRidge(
    globalPoints: Point[], 
    layerAssetId: string, 
    layerTransform: { x: number; y: number; scale: number; width: number; height: number }
  ): Promise<Point[]>;
  ```

- **Function:** Heal Patch isolated generation.
  ```typescript
  /**
   * Extracts texture strictly from the layer target, preserving transparent pixels properly.
   * @param layerData - The isolated ImageData of the target layer.
   * @param ridgeOriginalPath - The computed ridge in local layer coordinate space.
   * @param patchWidth - Average stroke width of the snatched line.
   * @returns The localized pixel buffer and its offset within the layer's local space.
   */
  public static async generateHealPatch(
    layerData: ImageData, 
    ridgeOriginalPath: Point[], 
    patchWidth: number
  ): Promise<{ buffer: Uint8Array, offset: Point }>;
  ```

By strictly dividing operations into isolated layer buffers, projecting transforms mathematically, and rigorously `revoking` object URLs, we guarantee a scalable, responsive UI that handles stacked high-fidelity OSM background data.

## 4. Testing Strategy & Unit Tests

To ensure the stability of the memory-constrained rasterization and hit-testing:

### 4.1 Required Unit Tests
*   **`osmImporter.test.ts`**:
    *   **Test Viewport Culling**: Mock `OffscreenCanvas`. Pass a mock `geoJSON` containing paths both inside and outside the `viewportBounds`. Assert that the 2D context `moveTo`/`lineTo` calls are ONLY generated for paths intersecting the bounds.
    *   **Test Max Zoom Limiting**: Verify that requesting a `targetZoom` greater than `MAX_RASTER_ZOOM_LEVEL` correctly caps the bounds and scaling factors before drawing.
*   **`useAppStore.test.ts` (Layer Management)**:
    *   **Test Zoom In Trigger**: Mock `currentGlobalScale` to advance past `rasterizedZoomLevel * 1.5`, verify `reRasterizeOSMLayer` is called with correct new scale and viewport parameters.
    *   **Test Zoom Out Memory Reclaim**: Mock `currentGlobalScale` to drop below `rasterizedZoomLevel * 0.5`, verify `reRasterizeOSMLayer` is triggered to down-sample.
    *   **Test Garbage Collection**: Assert that `URL.revokeObjectURL` is called with the old asset ID exactly once when a layer is re-rasterized or deleted.

### 4.2 Manual Verification
*   **Extreme Zoom Memory Profiling**: Open the app, import a large city OSM map. Zoom in continuously until hitting the max limitation lock. Monitor Chrome/Electron DevTools Memory tab to confirm the GPU memory stays within acceptable limits (~100-300MB) and does not crash the renderer process.
*   **Hit Test Latency Check**: Use the Extract tool on an imported layer. Ensure the `getImageData` 1x1 extraction strategy does not cause noticeable lag (should feel < 50ms) during the extraction click.
